## 1. 제품 개요

### 1.1 프로젝트 배경
일반 사용자들이 간편하게 일정을 관리할 수 있는 웹 기반 할일 관리 애플리케이션이 필요하다. 기존 복잡한 프로젝트 관리 도구와 달리, 누구나 쉽게 사용할 수 있는 직관적인 인터페이스를 제공하여 개인 생산성을 향상시킨다.

### 1.2 핵심 가치 제안
- **간편한 일정 관리**: 회원가입 후 바로 사용 가능한 직관적 UI
- **안전한 데이터 관리**: 사용자 인증을 통한 개인 데이터 보호
- **접근성**: 데스크톱/모바일 모두에서 사용 가능한 반응형 디자인

### 1.3 성공 지표
- 금요일 오후까지 MVP 출시 완료
- 핵심 기능(회원가입, 로그인, 할일 CRUD) 100% 동작
- 로컬 환경에서 통합 테스트 통과

## 4. 기능 요구사항

### 4.2 사용자 관리 기능

#### F-001: 회원가입 (P0)
**설명**: 신규 사용자가 계정을 생성한다.

**Acceptance Criteria**:
- [ ] username, password, email 입력 필드 제공
- [ ] username 중복 시 "Username already exists" 에러 표시 (E-001)
- [ ] 이메일 형식 검증 실패 시 "Invalid email format" 에러 표시 (E-003)
- [ ] 비밀번호는 bcrypt로 암호화하여 DB 저장 (cost factor 10 이상)
- [ ] 성공 시 HTTP 201 Created 반환 및 자동 로그인 또는 로그인 화면 이동
- [ ] 반응형 UI (데스크톱/모바일)

**관련 BR**: BR-001, BR-002, BR-003
**관련 UC**: UC-001
**API**: `POST /api/auth/register`

---

#### F-002: 로그인 (P0)
**설명**: 등록된 사용자가 자격증명을 통해 인증한다.

**Acceptance Criteria**:
- [ ] username, password 입력 필드 제공
- [ ] 자격증명 실패 시 "Invalid credentials" 에러 표시 (E-002)
- [ ] 성공 시 JWT 토큰 발급 (유효기간 24시간)
- [ ] 토큰은 httpOnly 쿠키 또는 localStorage에 저장
- [ ] 성공 시 할일 목록 화면으로 리다이렉트
- [ ] 반응형 UI (데스크톱/모바일)

**관련 BR**: BR-004
**관련 UC**: UC-002
**API**: `POST /api/auth/login`

---

#### F-003: 로그아웃 (P0)
**설명**: 로그인한 사용자가 세션을 종료한다.

**Acceptance Criteria**:
- [ ] 로그아웃 버튼 클릭 시 클라이언트 토큰 삭제

## 5. 기술 스택 및 아키텍처

### 5.1 시스템 아키텍처
**3-Tier Architecture**

```
┌─────────────────────────────────────────┐
│         Presentation Layer              │
│      React 19 + TypeScript              │
│    (SPA, 반응형 디자인)                   │
└─────────────────┬───────────────────────┘
                  │ REST API (JSON)
                  │ JWT Authentication
┌─────────────────▼───────────────────────┐
│         Application Layer               │
│      Node.js + Express                  │
│   pg 라이브러리 (PostgreSQL 연동)         │
│   ⚠️ Prisma 사용 절대 금지!              │
└─────────────────┬───────────────────────┘
                  │ SQL Queries
                  │ (pg 라이브러리)
┌─────────────────▼───────────────────────┐
│          Data Layer                     │
│        PostgreSQL 17                    │
│   (users, todos 테이블)                  │
└─────────────────────────────────────────┘
```

### 5.3 중요 기술 제약사항

⚠️ **절대 금지**
- **Prisma ORM 사용 금지**: 반드시 **pg 라이브러리**를 사용하여 SQL 쿼리 직접 작성
- 배포 환경 설정 금지: 로컬 개발 환경에서만 동작 확인

✅ **필수 사용**
- pg (node-postgres) 라이브러리로 PostgreSQL 연동
- bcrypt로 비밀번호 해싱
- JWT로 인증 토큰 관리

### 5.4 데이터베이스 스키마

```sql
-- users 테이블
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,  -- bcrypt 해시 (60자)
  email VARCHAR(100) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- todos 테이블
CREATE TABLE todos (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  due_date DATE,
  is_completed BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 인덱스
CREATE INDEX idx_todos_user_id ON todos(user_id);
CREATE INDEX idx_todos_is_completed ON todos(is_completed);
```

## 6. 비기능적 요구사항

### 6.1 성능
| 항목 | 목표 | 비고 |
|------|------|------|
| **API 응답 시간** | < 1초 | 로컬 환경 기준 |

| **동시 사용자** | 1명 | MVP는 개발자 본인 테스트 |

*3일 개발 기간이므로 기본적인 성능만 보장*

### 6.2 보안
| 항목 | 요구사항 | 구현 방법 |
|------|----------|-----------|
| **비밀번호 보안** | 암호화 저장 필수 | bcrypt (cost factor 10+) |
| **인증** | 토큰 기반 인증 | JWT (유효기간 24시간) |
| **인가** | 소유권 검증 | userId 기반 접근 제어 |
| **SQL Injection 방어** | 파라미터화된 쿼리 | pg 라이브러리의 parameterized query 사용 |
| **XSS 방어** | 입력 검증 및 이스케이핑 | express-validator 사용 |

**관련 ADR**: ADR-001 (JWT), ADR-003 (bcrypt)

### 6.3 사용성
| 항목 | 요구사항 |
|------|----------|
| **반응형 디자인** | 데스크톱(1024px+) 및 모바일(320px+) 지원 |
| **접근성** | 기본 수준 (시맨틱 HTML, 키보드 네비게이션) |
| **에러 메시지** | 사용자 친화적인 한글 메시지 |
| **로딩 상태** | API 호출 중 로딩 스피너 표시 |

### 6.4 유지보수성
| 항목 | 요구사항 |
|------|----------|
| **코드 스타일** | TypeScript 컨벤션 준수 |
| **에러 처리** | 모든 API는 일관된 에러 응답 구조 |
| **환경 변수** | DB 연결 정보, JWT Secret 등 환경 변수로 관리 |
