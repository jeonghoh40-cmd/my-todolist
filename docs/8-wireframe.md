## 1. 화면 흐름도

```
                    [앱 시작]
                        |
                        v
                 ┌─────────────┐
                 │  회원가입   │ (신규 사용자)
                 │ /register   │
                 └──────┬──────┘
                        |
                        v
        ┌───────────────┴───────────────┐
        |                               |
        v                               v
  ┌──────────┐                   ┌──────────┐
  │  로그인  │ <-----------------│ 자동이동 │
  │ /login   │                   └──────────┘
  └────┬─────┘
       |
       | (인증 성공)
       v
  ┌─────────────────┐
  │  할일 목록      │
  │   /todos        │
  └────┬──────┬─────┘
       |      |
       |      +------------------+
       |                         |
       v                         v
  ┌──────────────┐        ┌──────────────┐
  │ 할일 추가/수정│        │ 삭제 확인     │
  │   (모달)      │        │ (다이얼로그)  │
  └──────────────┘        └──────────────┘
```

**네비게이션 경로**:
- 회원가입 → 로그인 → 할일 목록
- 로그인 → 할일 목록
- 할일 목록 → 로그아웃 → 로그인
- 할일 목록 ↔ 할일 추가/수정 모달
- 할일 목록 ↔ 삭제 확인 다이얼로그

**정상 흐름**:
1. 사용자가 Username 입력 필드 클릭 → 포커스, 파란색 테두리
2. Username, Password, Email 순차 입력
3. "회원가입" 버튼 클릭 → 로딩 스피너 표시 (0.5초)
4. 성공 시 → 화면 상단 녹색 알림 "회원가입 성공!" (2초)
5. 자동으로 로그인 화면으로 전환 (페이드 애니메이션)

**에러 처리**:
- **E-001 (중복 username)**:
  - Username 필드 아래 빨간색 텍스트: "이미 존재하는 사용자명입니다"
  - Username 필드 테두리 빨간색
  - Username 필드 자동 포커스
- **E-003 (이메일 형식 오류)**:
  - Email 필드 아래 빨간색 텍스트: "올바른 이메일 형식이 아닙니다"
  - Email 필드 테두리 빨간색

**접근성**:
- Tab 키로 필드 간 이동 가능
- Enter 키로 폼 제출
- 모든 입력 필드에 label 연결
- 에러 메시지는 aria-live로 스크린리더 지원

#### 인터랙션

**할일 추가**:
1. "[+] 할일 추가" 버튼 클릭 → 할일 추가 모달 표시
2. 모달에서 정보 입력 후 저장 → 목록 상단에 새 할일 추가 (페이드인 애니메이션)

**할일 완료 처리**:
1. 체크박스 클릭 → 즉시 체크 표시 (애니메이션)
2. 제목에 취소선 적용 (0.3초 transition)
3. 카드 배경색 회색으로 변경
4. API 응답 후 updatedAt 갱신

**할일 수정**:
1. "수정" 버튼 클릭 → 할일 수정 모달 표시 (기존 데이터 미리 채움)
2. 정보 수정 후 저장 → 목록에서 해당 카드 업데이트 (페이드 효과)

**할일 삭제**:
1. "삭제" 버튼 클릭 → 삭제 확인 다이얼로그 표시
2. "삭제" 확인 → 카드 페이드아웃 후 슬라이드업 (0.5초)
3. 목록에서 제거

**로그아웃**:
1. "로그아웃" 버튼 클릭 → localStorage 토큰 삭제
2. 로그인 화면으로 즉시 전환

**반응형 동작**:
- 화면 너비 768px 미만: 1열 레이아웃
- 화면 너비 768px ~ 1024px: 2열 레이아웃
- 화면 너비 1024px 이상: 3열 레이아웃 (선택)

6. "취소" 또는 X 버튼 클릭 → 확인 없이 모달 닫힘 (입력 데이터 버림)

3. "저장" 클릭 → 업데이트 → 목록 반영

- 제목 필수: 빈 값 시 에러 표시

- 설명 선택: 빈 값 허용
- 마감일 선택: 빈 값 허용, 과거 날짜 허용

**DatePicker 동작**:
- 달력 아이콘 클릭 → 날짜 선택기 팝업
- 날짜 선택 → 필드에 "YYYY-MM-DD" 형식 표시
- 오늘 날짜 강조
- 선택된 날짜 파란색 배경

// App.tsx
<Router>

      ))}
    </TodoGrid>

### 4.2 API 연동

```typescript
// api.ts
const API_BASE = 'http://localhost:3001/api';

export const authAPI = {
  register: (data: RegisterDTO) => fetch(`${API_BASE}/auth/register`, { ... }),
  login: (data: LoginDTO) => fetch(`${API_BASE}/auth/login`, { ... }),
};

export const todoAPI = {
  getTodos: () => fetch(`${API_BASE}/todos`, { headers: { Authorization: `Bearer ${token}` } }),
  createTodo: (data: CreateTodoDTO) => fetch(`${API_BASE}/todos`, { method: 'POST', ... }),
  updateTodo: (id: number, data: UpdateTodoDTO) => fetch(`${API_BASE}/todos/${id}`, { method: 'PUT', ... }),
  deleteTodo: (id: number) => fetch(`${API_BASE}/todos/${id}`, { method: 'DELETE', ... }),
  toggleTodo: (id: number) => fetch(`${API_BASE}/todos/${id}/complete`, { method: 'PATCH', ... }),
};
```

### 4.3 개발 우선순위

**Phase 1 (오전)**: 기본 컴포넌트 구현

- 회원가입/로그인 화면 (레이아웃 + 스타일)

**Phase 2 (오후)**: 할일 목록 및 모달
- Checkbox, Modal 컴포넌트
- 할일 목록 화면 (그리드 레이아웃)
- 할일 추가/수정 모달
- 삭제 확인 다이얼로그

**Phase 3 (저녁)**: 반응형 및 인터랙션
- 모바일 반응형 CSS
- 애니메이션 추가 (페이드, 슬라이드)
- 에러/성공 알림
